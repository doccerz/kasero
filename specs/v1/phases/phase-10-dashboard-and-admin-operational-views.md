# Phase 10 — Dashboard & Admin Operational Views

## Goal
Deliver the main operational views needed by the admin user.

## Objectives
- Build the dashboard and space detail view.
- Surface the most operationally important records first.
- Present contract and ledger data clearly.

## Tasks
### Dashboard
- Build dashboard queries that prioritize:
  1. overdue tenants
  2. tenants nearing due date
  3. vacant spaces
- Show all spaces with occupancy derived from contract state.

### Space Detail View
- Display basic space information.
- Show contracts table.
- Highlight the current active contract.
- Show historical contracts.

### Contract Ledger View
- Build ledger view endpoint and UI.
- Show payables, payments, fund summary, and current amount due.
- Make audit-visible states easy to inspect for administrators.

## Deliverables
- Admin dashboard
- Space detail page
- Contract ledger view

## Exit Criteria
- Admin users can quickly identify urgent accounts and vacancy status.
- Contract history and financial state are visible from the admin UI.
